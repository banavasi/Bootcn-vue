# Welcome to bootcn-vue

**Accessible, customizable Bootstrap 5 + Vue 3 components with a shadcn/ui-inspired CLI.**

This is the component library documentation for bootcn-vue. Browse the sidebar to explore all available components, examples, and usage patterns.

## What is bootcn-vue?

**bootcn-vue** is a modern component library that brings Bootstrap 5's design system to Vue 3 with a copy-paste approach. Unlike traditional libraries, components are added directly to your project via CLI, giving you full control and customization freedom.

## Features

- üé® **Bootstrap 5 Design System** - Beautiful, responsive components following Bootstrap conventions
- ‚ö° **Vue 3 Composition API** - Modern Vue 3 with `<script setup>` and TypeScript
- üì¶ **CLI-First** - Copy components directly into your project, not node_modules
- ‚ôø **Accessible** - Built with [Reka UI](https://reka-ui.com) primitives for ARIA compliance
- üéØ **Customizable** - Full control over styles and variants using class-variance-authority
- üìò **TypeScript Native** - Fully typed components and props
- üîß **Copy & Own** - Components live in your codebase for easy modification
- ‚ö° **Zero Runtime Overhead** - No wrapper library, just pure Vue components

## üöÄ Getting Started

### Prerequisites

Before you begin, ensure you have:

- **Node.js** >= 20.0.0
- **Vue 3** >= 3.5.0
- **Vite** (Required for project setup)
- A Vue 3 + Vite project

### Step 1: Initialize bootcn-vue

Run the init command in your Vue 3 project:

```bash
npx @bootcn-vue/cli@latest init
```

This command will:

1. ‚úÖ Detect your Vue 3 + Vite project
2. ‚úÖ Set up path aliases (`@/*`) in `tsconfig.json` and `vite.config.ts`
3. ‚úÖ Create `bootcn.config.json` configuration file
4. ‚úÖ Create `src/lib/utils.ts` with utility functions
5. ‚úÖ Create `src/components/ui/` directory for components
6. ‚úÖ Install required dependencies:
   - `bootstrap` - Bootstrap 5 CSS framework
   - `reka-ui` - Accessible Vue primitives
   - `class-variance-authority` - Type-safe variant styling
   - `clsx` - Conditional class names
   - `tailwind-merge` - Intelligent class merging
   - `@fortawesome/fontawesome-free` - Icon library

### Step 2: Add Components

Add components to your project with the `add` command:

```bash
# Add a specific component
npx @bootcn-vue/cli@latest add button

# Add multiple components
npx @bootcn-vue/cli@latest add button tooltip checkbox radio

# Interactive mode - select from list
npx @bootcn-vue/cli@latest add
```

Components are copied to `src/components/ui/{ComponentName}/` with:

- Component Vue file (`ComponentName.vue`)
- Type definitions and variants (`index.ts`)
- Tests (`ComponentName.spec.ts`)

### Step 3: Use in Your App

Import and use components in your Vue files:

```vue
<script setup lang="ts">
import { Button } from "@/components/ui/Button";
import {
  Tooltip,
  TooltipTrigger,
  TooltipContent,
} from "@/components/ui/Tooltip";
import { Checkbox, CheckboxGroup } from "@/components/ui/Checkbox";
import { RadioGroup, RadioYesNo } from "@/components/ui/Radio";
</script>

<template>
  <div>
    <Button variant="primary" size="lg">Click me!</Button>

    <Tooltip>
      <TooltipTrigger>
        <Button variant="outline-secondary">Hover for info</Button>
      </TooltipTrigger>
      <TooltipContent>This is a helpful tooltip</TooltipContent>
    </Tooltip>

    <Checkbox v-model="isChecked" label="Accept terms" />

    <CheckboxGroup
      v-model="selectedOptions"
      :options="[
        { label: 'Option 1', value: '1' },
        { label: 'Option 2', value: '2' },
      ]"
    />

    <RadioGroup
      v-model="selectedValue"
      :options="[
        { label: 'Choice A', value: 'a' },
        { label: 'Choice B', value: 'b' },
      ]"
    />

    <RadioYesNo v-model="yesNoValue" />
  </div>
</template>
```

## üõ†Ô∏è CLI Commands Reference

### `init` - Initialize Project

```bash
npx @bootcn-vue/cli init
```

Sets up your Vue 3 project with bootcn-vue configuration, utilities, and dependencies.

**What it does:**

- Creates `bootcn.config.json` with your project settings
- Adds path aliases to `tsconfig.json` and `vite.config.ts`
- Creates `src/lib/utils.ts` with the `cn()` utility
- Creates `src/components/ui/` directory
- Installs required dependencies

### `add` - Add Components

```bash
# Add specific components
npx @bootcn-vue/cli add button
npx @bootcn-vue/cli add button tooltip field-text

# Interactive selection
npx @bootcn-vue/cli add
```

**Available Components:**

#### UI Components

- `button` - Button component with all Bootstrap variants
- `tooltip` - Tooltip with trigger and content (Tooltip, TooltipTrigger, TooltipContent)
- `checkbox` - Checkbox components (Checkbox, CheckboxGroup, CheckboxGroupItem, CheckboxPrimitive)
- `radio` - Radio button components (RadioGroup, RadioGroupItem, RadioYesNo)
- `field-text` - Complete text input field with validation
- `field-password` - Password field with show/hide toggle
- `field-ssn` - Social Security Number field with masking

#### Form Primitives

- `input-root` - Form input container primitive
- `input-label` - Form label with tooltip support
- `input-field` - Basic input field primitive
- `input-password` - Password input primitive
- `input-error` - Error message display
- `input-help` - Help text display
- `input-masked` - Input with masking support
- `input-numeric-range` - Numeric range input

**What it does:**

- Copies component files to `src/components/ui/{ComponentName}/`
- Transforms imports from package imports to local imports
- Makes components ready to use immediately

### `remove` - Remove Components

```bash
# Remove specific components
npx @bootcn-vue/cli remove button

# Interactive selection
npx @bootcn-vue/cli remove

# Remove ALL bootcn-vue setup
npx @bootcn-vue/cli remove --all
```

**The `--all` flag:**

- Deletes all components in `src/components/ui/`
- Deletes `src/lib/utils.ts` (and `lib/` directory if empty)
- Deletes `bootcn.config.json`
- Shows commands to uninstall dependencies
- Includes safety confirmations

## üé® Design System

### RDS Spacing System

bootcn-vue uses the **RDS (Responsive Design System) spacing scale** instead of Bootstrap's numeric spacing:

| Class        | Value         | Usage Example                   |
| ------------ | ------------- | ------------------------------- |
| `space-xxxs` | 0.25rem (4px) | `mb-space-xxxs`, `p-space-xxxs` |
| `space-xxs`  | 0.5rem (8px)  | `mb-space-xxs`, `gap-space-xxs` |
| `space-xs`   | 1rem (16px)   | `mb-space-xs`, `py-space-xs`    |
| `space-sm`   | 1.5rem (24px) | `mb-space-sm`, `mt-space-sm`    |
| `space-md`   | 2rem (32px)   | `mb-space-md`, `px-space-md`    |
| `space-lg`   | 3rem (48px)   | `mb-space-lg`, `p-space-lg`     |
| `space-xl`   | 4rem (64px)   | `mb-space-xl`, `mt-space-xl`    |
| `space-xxl`  | 6rem (96px)   | `mb-space-xxl`, `py-space-xxl`  |
| `space-xxxl` | 8rem (128px)  | `mb-space-xxxl`, `p-space-xxxl` |

**Examples:**

```vue
<template>
  <!-- Use RDS spacing -->
  <div class="mb-space-sm px-space-md">Content</div>

  <!-- NOT Bootstrap numeric spacing -->
  <!-- ‚ùå Don't use: mb-3, px-4 -->
</template>
```

### Typography

Use RDS heading sizes and body text classes:

- `h1-small`, `h1-medium`, `h1-large`
- `h2-small`, `h2-medium`, `h2-large`
- `h3-small`, `h3-medium`, `h3-large`

## üèóÔ∏è Contributing: Creating New Packages

If you're contributing to bootcn-vue and want to create a new component package:

### Using the Module Generator

```bash
# In the bootcn-vue monorepo root
pnpm create:module
```

The interactive CLI will prompt you for:

- **Module name** (e.g., `alerts`)
- **Description** (e.g., "Accessible, customizable alert components")
- **Component name** (e.g., `Alert`)
- **Author name**

This creates:

```
packages/
‚îî‚îÄ‚îÄ your-module/
    ‚îú‚îÄ‚îÄ package.json
    ‚îú‚îÄ‚îÄ tsconfig.json
    ‚îú‚îÄ‚îÄ tsup.config.ts
    ‚îú‚îÄ‚îÄ vitest.config.ts
    ‚îú‚îÄ‚îÄ README.md
    ‚îú‚îÄ‚îÄ CHANGELOG.md
    ‚îî‚îÄ‚îÄ src/
        ‚îú‚îÄ‚îÄ index.ts
        ‚îú‚îÄ‚îÄ env.d.ts
        ‚îî‚îÄ‚îÄ YourComponent/
            ‚îú‚îÄ‚îÄ YourComponent.vue
            ‚îú‚îÄ‚îÄ YourComponent.spec.ts
            ‚îî‚îÄ‚îÄ index.ts
```

### After Creating a Module

1. **Install dependencies:**

   ```bash
   pnpm install
   ```

2. **Build the package:**

   ```bash
   pnpm turbo build --filter=@bootcn-vue/your-module
   ```

3. **Add Storybook story** in `apps/playground/src/stories/`:

   ```typescript
   // YourComponent.stories.ts
   import type { Meta, StoryObj } from "@storybook/vue3";
   import { YourComponent } from "../components/ui/YourComponent";

   const meta: Meta<typeof YourComponent> = {
     title: "Components/YourComponent",
     component: YourComponent,
     tags: ["autodocs"],
   };

   export default meta;
   type Story = StoryObj<typeof meta>;

   export const Default: Story = {
     args: {
       // Your props
     },
   };
   ```

4. **View in Storybook:**

   ```bash
   pnpm dev
   ```

5. **Write tests:**

   ```bash
   pnpm test:unit -- YourComponent.spec.ts
   ```

6. **Create a changeset:**
   ```bash
   pnpm changeset
   # Select the package, choose version bump, write description
   ```

See [AGENTS.md](https://github.com/banavasi/Bootcn-vue/blob/main/AGENTS.md) in the repository for detailed development guidelines.

## üí° Philosophy

bootcn-vue follows the **shadcn/ui approach**: components are copied into your project, giving you full ownership and customization freedom.

### Why Copy-Paste?

- ‚úÖ **Full Control** - Modify components however you need
- ‚úÖ **No Version Lock** - No breaking changes from library updates
- ‚úÖ **Tree-Shakeable** - Only bundle what you use
- ‚úÖ **Easy Debugging** - See the actual component code
- ‚úÖ **No Bloat** - No unused components in your bundle
- ‚úÖ **Learn by Reading** - Understand how components work

### Why Bootstrap + Vue 3?

We love Bootstrap's proven design system, but wanted modern Vue 3 tooling:

- Built with **Vue 3 Composition API** (`<script setup>`)
- Full **TypeScript** support with type inference
- Accessible foundation with **Reka UI** primitives
- Modern styling with **class-variance-authority**
- Bootstrap 5 styles with **SCSS** customization
